# UI/UX review fixes: inline-style extraction and shared component library

A sweep across 21 pages replaces inline `style={{}}` patterns with scoped CSS classes in globals.css and introduces two shared React components (StatusDot, TabsPill). The highest-impact changes: card border-radius corrected to 20px per DESIGN.md, "coming soon" cards switched from `opacity: 0.5` to semantic `card-muted` class, all `👤` emoji replaced with lucide `User` icons, dashboard spotlight gradient now uses the existing `spotlight-violet` class instead of a hardcoded `#1f4d7a` gradient, and the dead `components/navbar.tsx` is removed. The prior review's three concerns (title weight, gradient fallbacks, bubble radius) are all resolved.

**Watch for:** The new `TabsPill` component is exported but never imported anywhere (confirmed). The `.demo-logo-mark` gradient uses `var(--grad-violet)` and `var(--grad-magenta)` without fallback values, inconsistent with the fallback-equipped `ask-mode-card--*` variants in the same diff (confirmed). The graph page tier-badge refactor leaves an inline-styled inner dot that references `meta.color` from JS rather than using the newly-defined CSS classes (confirmed).

## High-level view

The extraction strategy is consistent: find a repeated inline `style={{}}` block, create a named CSS class in globals.css, replace the JSX attribute with `className`. This works for single-use layout wrappers (`.auth-callback-wrapper`, `.demo-loading-page`) and for multi-instance patterns (`.btn-sm`, `.btn-xs`, `.timeline-*`).

Two shared components are introduced. `StatusDot` is consumed in Dashboard and Integrations, replacing inline-styled dots. `TabsPill` wraps Radix Tabs primitives but has zero consumers anywhere in the codebase — dead code shipped speculatively.

The "coming soon" card treatment improves over `opacity: 0.5` by applying muted text color to headings and body while keeping the card background at full opacity, preserving contrast ratios that opacity-based dimming degrades.

## TabsPill: a component without consumers

No page imports `TabsPill`. The Integrations page was refactored to use `.tabs-underline` / `.tabs-underline-trigger` CSS classes (also new in this diff). The Inbox page still uses inline filter buttons. If the intent is to use TabsPill for the Ask page's mode tabs, those still render as custom `<button>` elements with `data-active` rather than Radix Tabs primitives — so wiring it up would require restructuring the Ask page's tab logic.

## Demo logo gradient: inconsistent fallback treatment

The `.ask-mode-card--violet` declaration from the same diff uses the defensive pattern:

```css
var(--grad-violet, #6a4cf5)
```

The `.demo-logo-mark` added in the same commit doesn't:

```css
background: radial-gradient(120% 120% at 25% 15%, #8b6bff 0%, var(--grad-violet) 45%, var(--grad-magenta) 100%);
```

Both variables resolve today (defined at `:root`), but the inconsistency means one path has self-healing behavior and the other doesn't. One-line fix.

## Graph tier-badge: half-refactored

The outer pill badge moved from a 10-property inline style to `.tier-badge .tier-badge--{tier}`:

```css
.tier-badge--normal {
  color: var(--green);
  background: color-mix(in srgb, var(--green) 14%, transparent);
  border: 1px solid color-mix(in srgb, var(--green) 35%, transparent);
}
```

Inside the badge, a 7px dot still renders with `style={{ width: 7, height: 7, borderRadius: "50%", background: meta.color }}`. The diff already introduces `.tier-legend-dot--normal/amber/red` classes used for the legend at the top of the same page. The inner badge dot could reuse those (with a size override to 7px), removing the last inline-style reference to `meta.color`.
